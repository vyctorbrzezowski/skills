# Audio-Native Writing Reference

## Visuals

Classify each image before writing:

- Decorative: skip it.
- Atmospheric: mention only if it sets audience, place, or mood.
- Product/screenshot: describe the user-visible state and why it matters.
- Diagram/architecture: explain the flow, entities, and conclusion.
- Evidence: state what the image proves and the key detail listeners need.

Good visual narration is short and purposeful. It does not enumerate every pixel.

## Tables

Do not read tables cell by cell. Extract:

- the comparison axis
- the row or column that matters most
- the outlier or trend
- the decision the table supports

Use exact numbers only when they change the conclusion.

## Charts

Describe the chart type only if helpful. Prefer:

1. what is being measured
2. the visible trend or contrast
3. the conclusion
4. one or two anchor values if needed

## Code And Commands

Read exact code only when syntax is the point or the snippet is short. Otherwise:

- explain intent
- summarize inputs/outputs
- read one representative command
- direct listeners to the written article for full copy-paste material

## Links

Use the linked text, not the raw URL. If a link is the main call to action, say the destination plainly, such as "the advanced setup guide."

## Section Timing

For player headings, use stable source headings. If the article has meaningful intro text before the first heading, map that intro to the article title or a concise "Introduction" label. Do not let the intro consume the whole article; stop at the first `h2` or `h3`.

## Voice Test Excerpt

For choosing a voice, use 25-40 seconds that contains:

- one direct explanatory paragraph
- one proper noun or technical term
- one transition
- one sentence with emotional texture

Avoid cold openings made only of title/byline/date.
